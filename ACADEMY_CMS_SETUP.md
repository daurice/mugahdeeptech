# Academy CMS and LMS Control Guide

The Academy now has one database-driven content layer for the General, Kids, and Business homepages. Public homepage sections are rendered from MySQL; PHP templates provide layout and escaping, not marketing content.

## Setup

Import schemas in this order after the existing `database.sql`:

```bash
mysql -u root -p mugahdeeptech < database/kids-academy.sql
mysql -u root -p mugahdeeptech < database/kids-certificates.sql
mysql -u root -p mugahdeeptech < database/business-academy.sql
mysql -u root -p mugahdeeptech < database/academy-cms.sql
mysql -u root -p mugahdeeptech < database/academy-content-controls.sql
mysql -u root -p mugahdeeptech < database/academy-attempts.sql
```

Local port 3307:

```bash
C:/xampp/mysql/bin/mysql.exe --host=127.0.0.1 --port=3307 --user=root --execute="source database/academy-cms.sql; source database/academy-content-controls.sql; source database/academy-attempts.sql" mugahdeeptech
```

All seed scripts are designed to be rerun safely.

## Managing homepage sections

1. Open `/admin/academy-cms-dashboard.php`.
2. Select **Site Sections**.
3. Choose `academy_home`, `kids_home`, or `business_home`.
4. Edit the section fields or assign media/avatar records.
5. Use arrows to change order and Toggle to hide/show the section.
6. For FAQs, testimonials, stats, or feature cards, open **manage components**.

The home routes `/academy/`, `/kids/`, and `/business/` use `includes/academy-cms-renderer.php` and immediately reflect saved content.

## Media and avatar reuse

Upload assets once in **Media Library**. Files are MIME-validated, randomized, limited to 20 MB, and stored in `assets/uploads/academy/`. Assign the resulting asset to any section, component, pathway, course card, avatar, or lesson block.

Create avatars in **Avatars & Animations**, choose Kids/Business/General, and assign an uploaded asset or a trusted Lottie URL. Assign avatars to heroes through Site Sections and to lessons through Lesson Content.

## Roles and security

- `admin` users can create, edit, toggle, reorder, and delete CMS records.
- `editor` users can sign in and manage Academy CMS content but cannot delete CMS records or enter non-CMS admin pages.
- All mutations use CSRF validation and prepared SQL statements.
- Uploads are MIME-checked and randomized.
- Frontend output is HTML-escaped.
- Child profiles remain minimal and no public child chat exists.

To create an editor, insert a user with a password generated by `password_hash()` and role `editor` in the existing `users` table.

## Production checklist

- Enable HTTPS and replace the default admin password.
- Set PHP upload and request limits consistently with the 20 MB application limit.
- Review uploaded media and links before publishing.
- Back up the database and upload directories.
- Define retention periods for learner records and quiz attempts.
- Keep child data minimal and customer/business data out of unapproved AI tools.
